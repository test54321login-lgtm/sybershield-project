# Fix 404/SyntaxError Issues in CyberShield API

## Problem Analysis

The issue is a "domino effect" where:
1. **Primary**: API returns 404 (route not found or deployment issue)
2. **Secondary**: Frontend tries to parse HTML error page as JSON → SyntaxError

### Root Causes Identified:

1. **In `js/utils.js`**: The `apiFetch` function doesn't check if response is OK before returning
2. **In `js/auth.js`**: Calls `response.json()` even when response status is 404 (HTML returned)
3. **Potential**: Missing defensive error handling throughout the fetch chain

## Fix Plan

### 1. Fix `js/utils.js` - Add Defensive Error Handling

Update the `apiFetch` function to:
- Check `response.ok` status
- Log actual error text when server returns non-OK status
- Throw meaningful error instead of passing invalid response up

**Location**: `js/utils.js` lines 91-109

### 2. Fix `js/auth.js` - Proper Error Handling

Update `handleSignup` and `handleLogin` to:
- Check `response.ok` BEFORE trying to parse JSON
- Handle errors gracefully without double-parsing

**Locations**: 
- `js/auth.js` lines 20-45 (handleSignup)
- `js/auth.js` lines 52-89 (handleLogin)

### 3. Verify Deployment

- Confirm Vercel deployment includes `api/auth/signup.js` and `api/auth/login.js`
- Test API endpoint with external tool (Postman/curl) to verify route exists
- Check vercel.json configuration (currently looks correct)

## Implementation Steps

1. Edit `js/utils.js` - Add error checking in `apiFetch`
2. Edit `js/auth.js` - Fix error handling in both auth functions
