# Fix Plan: Normal Static Website - No Special Routing

## Issues Identified

1. **SyntaxError: Unexpected token '<' at feed.js:1:1** - Browser receives HTML instead of JS
2. **ReferenceError: loadDailyTip is not defined** - Script failed to load
3. **Styles not loading** - CSS also affected

## Root Cause

The catch-all route in `vercel.json` intercepts ALL requests and redirects to `index.html`:
```json
{ "src": "/.*", "dest": "/index.html" }
```

This breaks static file serving (JS, CSS, images).

## User's Requirement

"i want normal website" - meaning a standard static site where files are served directly without special routing/caching/offline support.

## Solution

Replace the problematic Vercel routing with proper static file handling. For a normal static website:

1. **Update vercel.json** - Remove catch-all route, use proper static serving
2. **Test the website** - Verify JS, CSS load correctly

## Implementation Steps

1. Read current `vercel.json`
2. Replace with proper static file configuration (no catch-all, no caching, no offline)
3. Test by running `vercel dev` and checking browser console for errors
