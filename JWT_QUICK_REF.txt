HOW JWT TOKENS WORK
═════════════════════════════════════════════════════════════════════════════

User Signs Up/Logs In
  ↓
Server Creates JWT Token with User Data
  ↓
Frontend Stores Token in localStorage
  ↓
Token Sent with Every API Request
  ↓
Server Verifies Token Using JWT_SECRET
  ↓
If Valid: Request Proceeds
If Invalid/Expired: User Logs In Again

PASSWORD SECURITY
═════════════════════════════════════════════════════════════════════════════

Algorithm: Bcryptjs
Salt Rounds: 12
Plain Password: User enters "MyPassword123"
Hashed Password: $2a$12$...80_character_hash...
Storage: Only hash is stored in MongoDB
Verification: Hash of entered password compared to stored hash
Security: Even identical passwords produce different hashes

TROUBLESHOOTING
═════════════════════════════════════════════════════════════════════════════

Problem: "MONGODB_URI is not set"
Solution: Check .env.local exists in project root

Problem: "Cannot connect to MongoDB"
Solution: Verify IP whitelist in MongoDB Atlas, check credentials

Problem: "Email already registered"
Solution: That email is already used, try logging in or use different email

Problem: "Too many login attempts"
Solution: Wait 15 minutes, rate limiting protection is active

Problem: "Invalid or expired token"
Solution: Log in again, token is only valid for 7 days
