HOW AUTHENTICATION WORKS
═════════════════════════════════════════════════════════════════════════════

User Signup:
  1. User fills signup form
  2. Frontend validates input
  3. Send to /api/auth/signup
  4. Server validates again
  5. Hash password with Bcryptjs
  6. Store user in MongoDB
  7. Generate JWT token
  8. Return token to frontend
  9. Frontend stores in localStorage
  10. User is logged in!

User Login:
  1. User enters email & password
  2. Frontend validates input
  3. Send to /api/auth/login
  4. Server finds user in MongoDB
  5. Compare entered password with stored hash
  6. If match: Generate JWT token
  7. Return token to frontend
  8. Frontend stores in localStorage
  9. User is logged in!

API Requests:
  1. Frontend sends request with JWT token in header
  2. Server verifies token using JWT secret
  3. If valid: Request proceeds
  4. If invalid/expired: Return 401 (user must login again)

TROUBLESHOOTING
═════════════════════════════════════════════════════════════════════════════

"Cannot connect to MongoDB"
  Check: MongoDB Atlas dashboard
  Do: Verify cluster is running, check IP whitelist

"npm run test fails"
  Check: Your MongoDB credentials
  Do: Copy URI directly from Atlas connection string

"Signup not working"
  Check: Browser console (F12 → Console tab)
  Do: Look for error messages, run npm run test

"Can't log in"
  Check: MongoDB Atlas → cybershield → users collection
  Do: Verify user was created in database

"Token not saving"
  Check: Browser DevTools → Storage → localStorage
  Do: You should see "cs_token" and "cs_user"
